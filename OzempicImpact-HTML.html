3. Causal Analysis Setup

Potential endogeneity issues in estimating the treatment effect of Ozempic on the target population:

Omitted Variable Bias: If there are unobserved variables that affect both the likelihood of receiving Ozempic (treatment) and the outcome (smart_allowed), omitting these from the analysis could lead to endogeneity. For instance, if patients with more severe health conditions are both more likely to receive Ozempic and have higher healthcare costs, failing to control for health severity could bias the treatment effect.
Simultaneity: If the outcome variable (smart_allowed) simultaneously influences the probability of receiving the treatment (proc_code = "J3490"), this could lead to endogeneity. For instance, if patients with higher predicted reimbursements are more likely to be prescribed Ozempic because it's more cost-effective for them.
Measurement Error: If there is measurement error in the independent variables, this could lead to endogeneity. For example, if patient_short_zip is inaccurately recorded, this could distort the relationship between geography and treatment effects.
Selection Bias: There might be a non-random selection into the treatment group. For example, if healthier or wealthier patients are more likely to be prescribed Ozempic, there might be a systematic difference between the treatment and control groups that is not captured by the observed variables.

Strategy for addressing endogeneity:

Double Lasso Regression: It helps in variable selection and can potentially control for endogeneity by considering selected covariates based on their predictive power for the treatment and the outcome.

Initial Data Clean Up
  1. I first filtered for obese and hypertension claims and saved them in two different data-frames. Diagnosis codes starting with 'E66' indicates 'Obese' and those that start with 'I10' indicate 'hypertension'.
  2. I chose 2 categorical features 'patient_gender','visit_type' and 2 numerical features 'patient_age', 'patient_short_zip' as the covariates.
  3. I encoded the categorical variables and imputed the missing values by the most frequently occuring category.
  4. Imputed the numerical features by their respective means.
  5. Rows with 'Proc_Code' = 'J3490' indicate treatment group, while the rest belong to the control group.
  6. I chose 'smart_allowed' as the target variable.
Double-Lasso Regression
  • Stage 1: Estimate d(x) with lasso regression of d on x.

  1. By estimating d(x), we're calculating the propensity score, which predicts the likelihood of a patient being treated with Ozempic based on their characteristics (captured in the covariates).

  2. With the propensity score accounting for the differences in covariates between groups, any remaining differences in outcomes can be attributed to the treatment itself, rather than confounding variables.

  • Stage 2: Do a lasso of y on [d, d(x), x], with d(x) unpenalized.

  1. By including the estimated treatment effects without a penalty, the model specifically accounts for the confounding variables' impact on treatment, thereby isolating the effect of the treatment on the outcome (smart_allowed).

  2. This stage helps in refining causal inference.

# To compute partial penalty
class SelectiveRegularizationLinearRegression:
    def __init__(self, alpha, apply_penalty, tolerance=1e-4, max_iterations=1000):
        self.alpha = alpha
        self.tolerance = tolerance
        self.apply_penalty = apply_penalty
        self.max_iterations = max_iterations
        self.w = None

    def _predicted_values(self, X, w):
        return np.matmul(X, w)

    def _rho_compute(self, y, X, w, j):
        X_k = np.delete(X, j, 1)
        w_k = np.delete(w, j)
        predict_k = self._predicted_values(X_k, w_k)
        residual = y - predict_k
        rho_j = np.sum(X[:, j] * residual)
        return rho_j

    def _z_compute(self, X):
        return np.sum(X * X, axis=0)

    def fit(self, X, y):
        n_samples, n_features = X.shape
        if not self.w:
            self.w = np.zeros(n_features)
        z = self._z_compute(X)
        for iteration in range(self.max_iterations):
            max_step = 0
            for j in range(n_features):
                rho_j = self._rho_compute(y, X, self.w, j)
                w_j_old = self.w[j]
                if j == 0:
                    self.w[j] = rho_j / z[j]
                else:
                    if self.apply_penalty[j]:
                        if rho_j < -self.alpha * n_samples:
                            self.w[j] = (rho_j + self.alpha * n_samples) / z[j]
                        elif -self.alpha * n_samples <= rho_j <= self.alpha * n_samples:
                            self.w[j] = 0.
                        elif rho_j > self.alpha * n_samples:
                            self.w[j] = (rho_j - self.alpha * n_samples) / z[j]
                    else:
                        self.w[j] = rho_j / z[j]
                max_step = max(max_step, abs(self.w[j] - w_j_old))
            if max_step < self.tolerance:
                break

        return self

    def predict(self, X):
        if self.w is None:
            raise ValueError("Model is not fitted yet!")
        return self._predicted_values(X, self.w)

    def coeffs(self):
        print(f"\nTreatment Coefficient: {self.w[0]}")
